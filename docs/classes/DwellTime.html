DwellTime Class

Defined in: src/library/ACT_DwellTime.js:7
Module: DwellTime

This module is helping tracking dwelltime for seletect dome element. Following IAB definition, dwelltime for desktop will be tracked as describe below

  • Start counting when mouse in target element
  • Stop counting when mouse leave target element
  • If mouse leave and come back within 2 seconds, continues counting
  • If mouse leave for more than 2 seconds, stop counting and send tracking label
  • If mouse leave after 10 minutes, the track will count as 10 minutes
  • Maximum counting time is 10 minutes
    // This example will start track dwelltime for ACT_mpu element
    var target = document.getElementById('ACT_mpu');
    var mpuDwellTimeTracker = new ACT.DwellTime({
          'targetElement': target,
          'targetName': 'mpu'
       // some other custom config
    });

    // To stop tracking dwelltime
    mpuDwellTimeTracker.destroy();

Provided by the SecureDarla module.

Defined in src/library/ACT_AdobeEdgeBridge.js:87

This is the function that you call to send and call redirect tracks. It sends, calls and catalogues the interaction tracks for AutoEvent and yFPAD to read when creating the footprint. This complies with adinterax redirect tracking. This also uses the "cap" flashvar to decide on "how" to call the redirect. This fixes the pop up blocking issues in internet explorer.

Parameters:

  • compId String

    This is the name of the composition.

  • action_name String

    This is the name of the common label that will be output in the post execution stats.

  • action Object

    This is the "action" or short name of the redirect

  • special String

    This is used when you need to pass in a "special" param to the url such as, "&zipcode=94587"

Example:

ACT.AdobeEdgeBridge.trackR(compId, "clickTAG", "gereral");
